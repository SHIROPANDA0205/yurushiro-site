# ゆるしろサイト｜Web公開までの手順書

この資料は、ローカルで作ったポートフォリオサイトを  
**GitHubに載せて → Vercelで公開する**までに実際に行った作業をまとめたものです。

公開URL: https://yurushiro-site.vercel.app  
GitHub: https://github.com/SHIROPANDA0205/yurushiro-site

---

## 全体の流れ（地図）

```
① ローカルでサイト完成
    ↓
② 本番ビルドが通るか確認（npm run build）
    ↓
③ Gitでコードを保存（init → commit）
    ↓
④ GitHubにリポジトリ作成 → push
    ↓
⑤ Vercelに接続してデプロイ（Web公開）
    ↓
⑥ （任意）お問い合わせフォームをメール連携
```

ポイント: **パソコンで直しただけでは公開サイトは変わらない。**  
「修正 → git commit → GitHubへ push → Vercelが自動再公開」の順です。

## ④ GitHubに載せる

### 4-1. GitHubにログイン（CLI）

端末から操作する場合は GitHub CLI（`gh`）を使います。

**方法A: デバイス認証（ブラウザでコード入力）**

```bash
gh auth login --hostname github.com --git-protocol https --web
```

表示されたURLを開き、ワンタイムコードを入力します。  
（うまくいかない場合は方法Bへ）

**方法B: Personal Access Token（推奨・確実）**

1. https://github.com/settings/tokens?type=beta を開く  
2. Generate new token  
   - Token name 例: `yurushiro-site`  
   - 権限: 対象リポジトリの **Contents: Read and write**  
   - 新規リポジトリ作成もするなら、作成権限も付与  
3. 発行されたトークンでログイン  

```bash
gh auth login --with-token
# 貼り付けて Enter
```

> **注意:** トークンはパスワード同然です。チャットやSNSに貼った場合は、作業後に必ず Revoke（失効）してください。

### 4-2. リポジトリ作成

GitHub上で空のリポジトリを作成します。

1. https://github.com/new  
2. Repository name: `yurushiro-site`  
3. Public  
4. README / .gitignore / license は**追加しない**（空のまま）  
5. Create repository

### 4-3. push

```bash
git remote add origin https://github.com/SHIROPANDA0205/yurushiro-site.git
gh auth setup-git   # 認証をgitに渡す（初回）
git push -u origin main
```

成功すると次でコードが見られます。  
https://github.com/SHIROPANDA0205/yurushiro-site

## ⑤ VercelでWeb公開

### 5-1. ログイン

```bash
npx vercel login
# または
npx vercel whoami
```

ブラウザで承認します。

### 5-2. 本番デプロイ

フォルダ名に日本語やスペースがあるとプロジェクト名に使えないため、名前を指定します。

```bash
npx vercel --prod --yes --name yurushiro-site
```

これで次が行われます。

- Vercelプロジェクト作成  
- GitHubリポジトリ連携  
- 本番ビルド・公開  

公開URLの例:

- 本番エイリアス: https://yurushiro-site.vercel.app  
- デプロイ固有URL: `https://yurushiro-site-xxxx-yurushiro.vercel.app`

ダッシュボード: https://vercel.com

### 5-3. 以降の更新のしかた

```bash
# 修正後
git add -A
git commit -m "変更内容の説明"
git push origin main
```

GitHubにpushされると、Vercelが自動で再デプロイします（数分）。  
ローカル保存だけでは公開サイトは変わりません。

## ⑥ お問い合わせフォーム → メール連携

Contactは最初「送信したフリ」だけでした。  
**Formspree** 経由で `yurushiro.contact@gmail.com` に届くようにしました。

### 6-1. 受信メール作成

Gmailなどで受信用アドレスを新規作成します。  
例: `yurushiro.contact@gmail.com`

### 6-2. Formspreeでフォーム作成

1. https://formspree.io に受信メールで登録  
2. New Form を作成（届く先 = 上記Gmail）  
3. フォームURLを控える  
   - 例: `https://formspree.io/f/mwvgqqpj`  
   - Form ID = `mwvgqqpj`

### 6-3. 環境変数を設定

コードは `NEXT_PUBLIC_FORMSPREE_ID` を読みます（`src/lib/contact.ts`）。

**ローカル（`.env.local` ※Gitに上げない）**

```bash
NEXT_PUBLIC_FORMSPREE_ID=mwvgqqpj
```

**Vercel**

```bash
printf 'mwvgqqpj' | npx vercel env add NEXT_PUBLIC_FORMSPREE_ID production
printf 'mwvgqqpj' | npx vercel env add NEXT_PUBLIC_FORMSPREE_ID preview
printf 'mwvgqqpj' | npx vercel env add NEXT_PUBLIC_FORMSPREE_ID development
```

`NEXT_PUBLIC_` はビルド時に埋め込まれるため、**環境変数追加後は再デプロイが必要**です。

```bash
git add src/lib/contact.ts .env.example
git commit -m "Connect contact form to Formspree for email delivery"
git push origin main
# 必要なら明示的に再デプロイ
npx vercel --prod --yes
```

### 6-4. 動作確認

1. https://yurushiro-site.vercel.app のContactからテスト送信  
2. Gmailに届くか確認  
3. 初回はFormspreeの確認メールが来る場合あり → リンクを開いて有効化

## よくあるつまずき

| 症状 | 原因と対処 |
| --- | --- |
| `gh auth login` のデバイス認証が失敗する | Personal Access Token方式に切り替える |
| `createRepository` 権限エラー | トークンにリポジトリ作成権限がない → GitHubサイトで手動作成 |
| `Permission denied` / 403 on push | トークンにそのリポの **Contents: Read and write** がない |
| Vercelプロジェクト名エラー | 日本語フォルダ名が原因 → `--name yurushiro-site` を指定 |
| ローカルで直したのにサイトが変わらない | pushしていない。commit → push が必要 |
| フォームが送れない / 届かない | Form ID未設定、Vercel再デプロイ漏れ、Formspree未確認 |
